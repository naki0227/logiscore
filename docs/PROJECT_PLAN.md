# Logiscore — プロジェクト企画書

> **プロジェクト正式名称:** Logiscore (旧称: Enludus: Harmonic Entropy)
> **作成日:** 2026-03-23
> **バージョン:** 1.1.0

---

## 1. エグゼクティブサマリー

Logiscore は、**ソースコードを音楽（MIDI）へ変換し、音楽からソースコードを完全復元する**可逆エンコーディングシステムである。

プログラミング言語ごとに異なる音階・音色で演奏されることで、コードが「聴ける」体験を提供する。単なるギミックではなく、**データの可逆圧縮とMIDIプロトコルの精密な制御**を基盤とした技術デモであり、クリエイティブコーディングの新境地を切り開く。

---

## 2. 課題と解決策

### 2.1 課題

| # | 課題 | 詳細 |
|---|------|------|
| 1 | コードの「不可視性」 | ソースコードは視覚的なテキストに過ぎず、感覚的に体験する手段がない |
| 2 | データエンコーディングの教育 | ビット操作・圧縮・プロトコル設計を体感的に学ぶツールが不足 |
| 3 | クリエイティブコーディングの限界 | code → visual は多いが code → audio の可逆変換は未踏 |

### 2.2 解決策

Logiscore は以下の3つの価値を提供する：

1. **可逆変換エンジン（Rust/WASM）**: ソースコードをzlib圧縮し、独自プロトコルでMIDIデータに変換。逆変換で1bitの欠落もなく復元。
2. **言語ごとの音楽的個性**: 拡張子に応じた音階（Major/Minor/Industrial等）を割り当て、言語の「性格」を音で表現。
3. **シンセサイザー風UI**: 本物のハードウェアシンセを模した操作パネルで、直感的にエンコード/デコード/再生を体験。

## 4. コア機能

### 4.1 エンコード（Code → Music）

```
[Source Code]
    ↓ flate2 (zlib圧縮)
[Compressed Bytes]
    ↓ Header Byte 付与 (Scale ID + Root Key)
[Protocol Bytes]
    ↓ 16ch × 3和音 パッキング
[MIDI Binary (SMF)]
    ↓ Tone.js
[♪ 再生]
```

### 4.2 デコード（Music → Code）

```
[MIDI Binary]
    ↓ midly パース
[Note Events per Channel/Tick]
    ↓ Pitch Offset + Velocity → Byte 復元
[Protocol Bytes]
    ↓ Header 除去 + flate2 展開
[Source Code]  ← 完全一致
```

### 4.3 UI/UX

- **Monaco Editor**: コード入力エリア（シンタックスハイライト付き）
- **シンセパネル**: ノブ（Root Key選択）、スイッチ（Scale切替）、再生/停止ボタン
- **ビジュアライザー**: 再生中のノートに対応するコード行がリアルタイムでハイライト

## 5. 技術スタック概要

| レイヤー | 技術 | 役割 |
|----------|------|------|
| **Core Engine** | Rust | 圧縮・プロトコル変換・MIDI生成 |
| **Bridge** | wasm-bindgen | Rust ↔ JS 間のバイナリ転送 |
| **Frontend** | Vite + React (SPA) | UI・状態管理 |
| **Audio** | Tone.js | MIDIデータのリアルタイム再生 |
| **Editor** | Monaco Editor | コード入力・ハイライト |
| **Animation** | Framer Motion | シンセUI のインタラクション |
| **Visualization** | Canvas 2D | 波形・ノートの可視化 |

## 6. ビジネス的位置づけ

### 6.1 KPI（技術デモとして）

| 指標 | 目標 |
|------|------|
| **可逆性** | 100% (1bitの差分もなし) |
| **対応言語** | 5種類以上 (.rs, .ts, .py, .go, .json) |
| **最大入力サイズ** | 100KB のソースコードを安定処理 |
| **WASM初期化時間** | < 500ms |
| **GitHub Stars** | ポートフォリオとしての技術的訴求力 |

### 6.2 リスク分析

| リスク | 影響 | 軽減策 |
|--------|------|--------|
| WASMメモリ不足 | 大ファイルで変換失敗 | チャンク分割処理の実装 |
| MIDIフォーマット互換性 | DAWで再生不可 | SMF Type 0/1 両対応 |
| ブラウザ互換性 | WebGPU非対応環境 | Canvas 2Dへのフォールバック |
